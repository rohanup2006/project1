# 🎓 Student Marks Analyzer

A 3-tier software application built to clean flat files, structure relational tables, and present real-time student performance metrics.

---

## 📌 Project Goals
- Automates tedious data cleaning loops.
- Centralizes scattered academic Excel spreadsheets.
- Replaces slow, static recording systems.
- Prevents redundant student record duplication.
- Filters performance trends across multiple classrooms.

---

## 🏗️ System Components

### 1. Ingestion Engine
- Automatically parses local offline CSV resources.
- Strips out messy trailing text whitespaces.
- Normalizes conflicting human-entered date styles.
- Maps structural variables for database alignment.
- Safeguards live pipelines from row-loading faults.

### 2. SQL Server DB (SSMS)
- Deploys clean relational storage tables.
- Organizes student biological rows securely.
- Anchors master course lists uniquely.
- Binds test metrics via Foreign Keys.
- Leverages transaction rollbacks for safety.

### 3. Streamlit Interface
- Forms direct network socket handshakes.
- Queries backend records using data frames.
- Builds persistent dropdown filter panels.
- Calculates overall cohort percentage metrics.
- Renders responsive bar chart averages.

---

## 💡 Tech Summary
- **Data Cleaner**: Pandas & Python
- **Database Engine**: Microsoft SQL Server
- **Server Adapter**: pyodbc Driver
- **Presentation UI**: Streamlit Platform
- **Format Target**: Relational Database

## 🚀 Execution Steps

### 1. Database Creation
- Connect to local SQL Server instance.
- Execute table generation database schemas.
- Assign appropriate integer matching index lengths.
- Establish baseline entity constraint limits.
- Clear workspace for external pipeline testing.

### 2. File Placement
- Create standard working directory folders.
- Store original student record flat files.
- Gather separate baseline core course sheets.
- Collect calculated individual grade breakdown logs.
- Match all dataset naming parameters exactly.

### 3. Running Migration
- Execute local data import script files.
- Monitor active automated staging cleanup processes.
- Ensure all datatypes match destination layouts.
- Verify safe atomic multi-row commit passes.
- Inspect primary keys for structural issues.

### 4. Dashboard Launch
- Initialize core browser UI rendering engines.
- Connect interface layout settings to local hosts.
- Verify running state of page connections.
- Test active interactive dropdown filter widgets.
- View real-time database metric calculation results.
